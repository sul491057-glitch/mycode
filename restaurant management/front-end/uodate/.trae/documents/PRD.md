# 餐厅预订综合业务管理系统 - 产品需求文档 (PRD)

## 1. 项目概述
本项目旨在开发一套餐厅预订与业务管理系统，分为**顾客端**和**管理员端**。系统需支持响应式布局（适配移动端和PC端），提供直观流畅的用户操作体验。

## 2. 用户角色
### 2.1 顾客 (Customer)
- **登录方式**：免密登录。在登录页面点击“用户登录”按钮直接跳转至用户主页。
- **权限**：
  - 查看商品列表
  - 查看推荐菜品
  - 下单点餐
  - 预定餐桌

### 2.2 管理员 (Admin)
- **登录方式**：密码登录。需输入账号和密码。
- **权限**：
  - 查看顾客下单的餐品列表
  - 手动设置/推荐商品

## 3. 功能需求

### 3.1 登录模块
- **页面布局**：
  - 顶部/中心：系统标题/Logo。
  - 中部：管理员登录表单（用户名、密码输入框、登录按钮）。
  - 底部：显著的“用户登录”按钮（样式需与管理员登录按钮保持一致或协调），点击直接进入顾客首页。
- **交互**：
  - 表单验证（管理员登录）。
  - 响应式适配。

### 3.2 顾客端功能
- **首页/推荐页**：
  - 展示管理员推荐的菜品（轮播图或置顶展示）。
  - 展示热门/促销信息。
- **商品列表页**：
  - 分类展示所有菜品。
  - 支持加入购物车/直接下单。
- **下单流程**：
  - 购物车确认 -> 提交订单 -> 模拟支付/下单成功反馈。
- **餐桌预订**：
  - 选择日期、时间、人数。
  - 提交预订申请。

### 3.3 管理员端功能
- **订单管理**：
  - 列表展示所有顾客提交的订单。
  - 查看订单详情（菜品、金额、桌号/预订信息）。
- **推荐管理**：
  - 商品列表管理。
  - “设为推荐”/“取消推荐”开关。

## 4. 非功能需求
- **UI/UX**：
  - 使用 Element Plus 组件库。
  - 界面语言：全中文。
  - 响应式布局：兼容 PC 和 移动端（使用 Grid 或 Flex 布局）。
- **数据**：
  - 前端通过 API 获取数据。
  - 开发阶段使用 Mock 数据模拟后端响应。
